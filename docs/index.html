agentliveness

Your scheduled agent can be dead for weeks while every check says it's fine.

A small, dependency-free reliability harness for scheduled / autonomous agent systems. It persists state across runs, proves liveness — not just freshness — fails loud, and scans for the failure class before it ships.

See the failure

Under a scheduler — cron, launchd, a Kubernetes CronJob — every run is a fresh process. A counter initialized in __init__ and held only in memory re-seeds to its starting value on every run. In a long-lived process it accumulates and everything looks right. In production it resets forever — and the output keeps getting written, so freshness checks pass with flying colors.

Freshness is not liveness. A fresh file over a dead producer passes every freshness check and fails the only one that matters.

The harness

Four composing primitives. The first three run in production; the fourth shifts left and catches the failure statically.

# 1 · persist — state that survives process death
from agentliveness import PersistentState
st = PersistentState("~/.myagent-state.json", default={"runs": 0})
d = st.load(); d["runs"] += 1; st.save(d)   # atomic; survives restart

# 2 · detect — producing, not merely fresh
from agentliveness import LivenessContract
c = LivenessContract(path="~/.myagent.json", max_age_s=2*3600,
                     producing=lambda p: bool(p.get("norms")))
if not c.evaluate().healthy: alert(...)   # "fresh but EMPTY"

# 3 · respond — fire once per incident, never crash the run
from agentliveness import LoudFail, log_sink, notify_sink
LoudFail("my-agent", "~/.my-agent-loudfail.json",
         sinks=[log_sink(), notify_sink()]).report(c.evaluate())

Catch the whole class before it ships

agentliveness audit doesn't run your code — it reads its shape and looks for three structural signs that together are the failure class. No execution, no scheduler, no waiting weeks.

agentliveness audit path/to/agent        # scan a file or directory
agentliveness audit . --json             # machine-readable
agentliveness audit . --exit-zero        # report only; don't fail CI

Why this matters more, not less

We're about to run more of the world on schedules, unattended, with agents in the loop. The dangerous failures won't be loud crashes — those you'll see. They'll be fresh timestamps over dead engines: outputs that look alive, checks that pass, and behaviour that quietly stopped working weeks ago. Predict less. Detect more.
